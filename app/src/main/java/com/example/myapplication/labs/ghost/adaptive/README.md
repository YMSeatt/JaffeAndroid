# 👻 Ghost Adaptive UI: Density-Aware Layouts

Ghost Adaptive UI is an experimental system that analyzes student density in the classroom to optimize spatial flow and maintain pedagogical balance.

## 🛰️ The Density Model
The system uses a **20x20 grid** mapped across the 4000x4000 logical canvas. Student positions are aggregated into this grid, creating a "Social Pressure" map.

- **Crowding Zones**: Areas with high student density (> 0.7 normalized).
- **Low-Pressure Zones**: Empty or sparse areas of the classroom.

## 🎨 Visualization (AGSL)
The density map is visualized using a high-performance **AGSL Shader**.
- **Amber Fields**: Represent high-density "Heat Zones" that may require teacher attention or layout adjustment.
- **Cyan Fields**: Represent balanced or sparse areas.
- **Pulsation**: The visual fields pulse based on density, providing a living representation of classroom energy.

## 🏗️ Adaptive Engine
The `GhostAdaptiveEngine` provides two core functions:
1. **Metrics Calculation**: Transforms student coordinates into normalized density zones.
2. **Layout Proposals**: Suggests moving students from high-pressure zones to the nearest low-pressure neighbor to alleviate crowding.

## 🔄 Logic Parity
This implementation is designed to align with theoretical models of "Social Fluidity" and spatial heuristics. It compensates for Android's 4000x4000 coordinate system by using a localized grid-blur algorithm to ensure smooth density transitions.
