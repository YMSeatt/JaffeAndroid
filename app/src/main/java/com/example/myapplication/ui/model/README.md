# 💎 UI Models & Reactive State Containers

This package defines the UI-specific data models used by the Jetpack Compose layer. These models act as reactive bridges between the relational Room entities and the interactive seating chart.

## 🏛️ The "Fluid Interaction" Performance Model

To achieve a buttery-smooth 60fps experience—especially during high-frequency gestures like dragging student groups or resizing items—the UI models utilize a specialized performance architecture:

1.  **MutableState Properties**: Unlike standard data classes or Room entities, the primary fields in `StudentUiItem` and `FurnitureUiItem` are wrapped in Compose `MutableState`. This allows for fine-grained observation; Compose only recomposes the specific UI nodes (e.g., just the (X,Y) coordinates or just the background color) that change, rather than the entire icon or canvas.
2.  **Optimistic UI Updates**: During interactive gestures (drag-and-drop), components directly update the `MutableState` properties of these UI items. This provides immediate visual feedback to the user by bypassing the database-to-UI update loop.
3.  **Identity Preservation**: The `SeatingChartViewModel` maintains a persistent cache of these UI items. When data changes in the database, the repository updates the *existing* fields of these cached items rather than creating new objects. This preserves object identity, which is critical for Compose's internal "diff-and-patch" optimization.

## 👻 Ghost Lab Integration

These models also serve as the storage containers for advanced R&D metrics generated by the "Ghost Lab" engines:
-   **`behaviorEntropy`**: Pre-calculated by the `GhostEntropyEngine` to drive neural visualizations.
-   **`tectonicStress`**: Pre-calculated by the `GhostTectonicEngine` to visualize social stability.
-   **`irisParams`**: Procedural parameters for student-specific fractal signatures.
-   **`osmoticNode`**: Spatial data for knowledge diffusion simulations.

## 📂 Key Components

-   **`StudentUiItem`**: The reactive container for student state on the seating chart.
-   **`FurnitureUiItem`**: The reactive container for desks, tables, and other classroom objects.
-   **`StudentExtensions.kt`**: Contains the mapping logic (`toStudentUiItem`) and the differential sync logic (`updateStudentUiItem`) that powers the performance pipeline.
-   **`Mappers.kt`**: Handles the transformation of Furniture entities into UI models.
